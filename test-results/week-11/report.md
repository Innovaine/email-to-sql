## Root cause analysis

The week-11 changes to `src/extraction.ts` modified the confidence scoring formula. Looking at the pattern:
- High-confidence fixture now passes (good)
- Medium-confidence fixture scores too high (0.84 vs <0.8 expected)
- Low-confidence fixture scores too high (0.81 vs <0.65 expected)

The confidence boost is too aggressive. It's pushing marginal-quality extractions over the auto-sync threshold (0.75), which means poor leads will pollute the Pipedrive pipeline without manual review.

The company name extraction also regressed - instead of capturing "TechRX" it's picking up "a healthcare tech co" (descriptor phrase instead of proper name).

## What to fix next round

**Immediate blockers (before production):**
1. Revert or tune the confidence scoring changes in extraction.ts
   - The formula is inflating scores by ~10-15 points
   - Need to test against ALL fixtures, not just the high-confidence one
   
2. Fix company name extraction regex/logic
   - Prioritize proper nouns over descriptive phrases
   - Test case: "I'm from TechRX, a healthcare tech co" should extract "TechRX"

3. Add input validation for malformed emails
   - Return 400 Bad Request when From: or Subject: headers are missing
   - Don't attempt extraction on invalid input

4. Fix the dashboard test selector
   - Use `.first()` or more specific selector to avoid strict mode violation

**Testing approach for week 12:**
- Run the FULL test suite before considering any bug "fixed"
- Week 11 shows fixing one test can break three others
- All 28 tests must pass before claiming production-ready

**User impact right now:**
Core functionality works - users can submit emails, see dashboard, get extractions. But the confidence scoring is broken, which means:
- Ambiguous emails auto-sync to Pipedrive instead of queuing for review
- Sales team sees low-quality leads in their pipeline
- CRM data quality degrades over time

This product should NOT ship until confidence scoring is reliable. A "high confidence" score must actually mean high confidence, or the auto-sync feature becomes a liability instead of a time-saver.
